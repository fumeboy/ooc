# 调试指南

## 方式一：使用 Debug 标志（推荐）

### 启动 Debug 模式

```bash
# 启用 debug 模式，输出详细日志
go run cmd/server/main.go -debug
```

或者指定其他参数：

```bash
go run cmd/server/main.go -debug -config .conf.xml -port 8080
```

### Debug 模式特性

- 启用详细的 HTTP 请求日志
- 显示请求方法、URI、状态码、延迟时间
- 显示错误信息

## 方式二：使用 Delve 调试器（断点调试）

### 安装 Delve

```bash
go install github.com/go-delve/delve/cmd/dlv@latest
```

验证安装：

```bash
dlv version
```

### 使用 Delve 启动调试

#### 方法 1：直接调试（推荐）

```bash
# 在项目根目录下
dlv debug cmd/server/main.go -- -config .conf.xml -port 8080
```

#### 方法 2：先编译再调试

```bash
# 编译（带调试信息）
go build -gcflags="all=-N -l" -o bin/server cmd/server/main.go

# 使用 dlv 调试
dlv exec bin/server -- -config .conf.xml -port 8080
```

### Delve 常用命令

启动后，你会进入 Delve 的交互式调试界面：

```
Type 'help' for list of commands.
(dlv) 
```

#### 基本调试命令

```bash
# 设置断点
(dlv) break main.main          # 在 main 函数设置断点
(dlv) break server.GetSession   # 在 GetSession 函数设置断点
(dlv) break cmd/server/main.go:49  # 在指定文件的行号设置断点

# 继续执行
(dlv) continue                 # 或 c

# 单步执行
(dlv) next                     # 或 n - 执行下一行（不进入函数）
(dlv) step                     # 或 s - 执行下一行（进入函数）
(dlv) stepout                  # 跳出当前函数

# 查看变量
(dlv) print variableName       # 或 p - 打印变量值
(dlv) locals                   # 查看所有局部变量
(dlv) args                     # 查看函数参数

# 查看调用栈
(dlv) stack                    # 或 bt - 查看调用栈
(dlv) goroutine                # 查看当前 goroutine

# 查看源码
(dlv) list                     # 显示当前代码
(dlv) list 50                  # 显示第 50 行附近的代码

# 运行到指定位置
(dlv) continue                 # 继续执行直到下一个断点
(dlv) restart                  # 重新启动程序

# 退出调试
(dlv) exit                     # 或 quit
```

#### 示例调试会话

```bash
# 1. 启动调试
dlv debug cmd/server/main.go -- -port 8080

# 2. 设置断点
(dlv) break main.main
(dlv) break server.GetSession

# 3. 开始执行
(dlv) continue

# 4. 程序会在 main.main 处暂停，查看变量
(dlv) print *configPath
(dlv) print *port

# 5. 继续执行
(dlv) continue

# 6. 当有请求到达时，会在 GetSession 处暂停
# 可以查看请求参数、Session 数据等
(dlv) print sessionID
(dlv) locals
```

## 方式三：使用 VS Code 调试

### 配置 launch.json

在项目根目录创建 `.vscode/launch.json`：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug Server",
      "type": "go",
      "request": "launch",
      "mode": "debug",
      "program": "${workspaceFolder}/cmd/server",
      "args": [
        "-config", ".conf.xml",
        "-port", "8080",
        "-debug"
      ],
      "env": {},
      "showLog": true
    }
  ]
}
```

### 使用步骤

1. 在 VS Code 中打开项目
2. 按 `F5` 或点击"运行和调试"
3. 选择 "Debug Server" 配置
4. 设置断点（点击行号左侧）
5. 开始调试

## 方式四：使用环境变量控制日志

### 设置日志级别

```bash
# 启用详细日志
export OOC_DEBUG=true
go run cmd/server/main.go
```

### 在代码中使用

可以在代码中检查环境变量：

```go
if os.Getenv("OOC_DEBUG") == "true" {
    log.SetFlags(log.LstdFlags | log.Lshortfile)
    // 启用详细日志
}
```

## 调试技巧

### 1. 查看 HTTP 请求详情

在 Debug 模式下，Echo 的 Logger 中间件会输出详细的请求信息：
- 请求方法
- 请求 URI
- 响应状态码
- 处理时间
- 错误信息

### 2. 调试 LLM 调用

在 `internal/client/llm/http_client.go` 中添加日志：

```go
if *debug {
    log.Printf("LLM Request: %s", string(payload))
    log.Printf("LLM Response: %s", string(body))
}
```

### 3. 调试思考循环

在 `internal/agent/thinkloop.go` 的 `Think` 方法中添加日志：

```go
if *debug {
    log.Printf("Think Loop - Conversation ID: %s", conv.ID)
    log.Printf("Think Loop - Prompt: %s", req)
}
```

### 4. 使用 pprof 进行性能分析

添加性能分析端点：

```go
import _ "net/http/pprof"

// 在 main 函数中添加
if *debug {
    go func() {
        log.Println("pprof server started on :6060")
        log.Println(http.ListenAndServe("localhost:6060", nil))
    }()
}
```

然后可以访问：
- `http://localhost:6060/debug/pprof/` - 性能分析主页
- `http://localhost:6060/debug/pprof/heap` - 堆内存分析
- `http://localhost:6060/debug/pprof/profile` - CPU 性能分析

## 常见问题

### 1. Delve 无法附加到进程

确保编译时包含调试信息：

```bash
go build -gcflags="all=-N -l" -o bin/server cmd/server/main.go
```

### 2. 断点不生效

- 确保代码已保存
- 确保断点设置在可执行代码行（不是注释或空行）
- 尝试重新编译

### 3. 无法查看变量值

- 确保使用 `-gcflags="all=-N -l"` 编译
- 变量可能已被优化掉，尝试禁用优化

### 4. 调试时程序运行缓慢

这是正常的，调试模式下程序会变慢。如果只是查看日志，使用 `-debug` 标志即可。

## 推荐工作流

1. **开发阶段**：使用 `-debug` 标志查看详细日志
2. **问题排查**：使用 Delve 设置断点进行深入调试
3. **性能分析**：使用 pprof 分析性能瓶颈
4. **生产环境**：关闭 debug 模式，使用简洁日志
