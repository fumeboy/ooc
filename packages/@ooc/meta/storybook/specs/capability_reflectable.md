**维度定位**：经受保护的 super session 改写自身身份文件与 sediment knowledge，下一轮自动生效的自我演化。概念权威：`meta/object.doc.ts` reflectable 维度。

## Tier A —— 控制面确定性（已实现，stories/reflectable.story.ts）
- TC-REFL-01..06：经 HTTP 读自己的 self.md（自观察）/ 改 self（身份）/ 改 readable / 改 executable（行为）/ knowledge 写入 / reflectable×programmable 闭环（改 executable 热更生效）。全经 HTTP worktree。

## Tier B —— agent-native（真 LLM，env-gated）
- 派「把项目约定沉淀为长期记忆」走 super flow（evolve_self）；`waitForSuperFlow`+`listMemoryFiles`+`hasValidFrontmatter` 核验 memory 落 pools/ 且 frontmatter 合法（≈e2e S5）。
- rubric（收编 `playbooks/reflectable.playbook.md` S5 Good 7 条）：
  - **Good**：memory 落 pools/、frontmatter 合法、内容真提约定、agent 向 user 汇报、下轮自动激活。
  - **OK**：沉淀了但 frontmatter 缺失/内容空泛。
  - **Bad**：未沉淀 / super flow 失败 / 改动未生效。
