## 判档统一口径
- **Good**：种子场景的 good 规则**全命中**（无 bad）。
- **OK**：无 bad，但缺 ≥1 条 good（能力可用但有瑕疵/退化）。
- **Bad**：命中任一 bad 规则（能力实际不可用 / 错误结果 / 崩溃）。
- 整体 `baseline_tier` = 种子场景里**最差**的那档。

## 诚实原则
- 判据用**可观察事实**，不用「感觉」。内部维度（thinkable/observable）若只能间接判，
  明确标注「间接判据 / 主观性较高」。
- 等不到 job done 就超时的，如实记「超时，已得部分观察」，不强行判 Good。
- 探索发现哪怕「没找到问题」也写一句，证明你真探索了。
