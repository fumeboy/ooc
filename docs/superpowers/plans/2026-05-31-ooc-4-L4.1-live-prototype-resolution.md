# OOC-4 L4.1：活路径 prototype-chain 解析机制 + skill_index 端到端（renderXml + basicKnowledge）

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development / executing-plans。
> **执行纪律**：执行 sub-agent **不要自己 commit**——跑 tsc/bun test 到全绿，Supervisor 整合提交。

**Goal:** 把 window 的 **renderXml + basicKnowledge** 两个 aspect 从 `getWindowTypeDefinition(window.type)` 改为「沿 base prototype 链解析、registry 兜底」，用最简原型 **skill_index** 端到端证明：skill_index 的 renderXml/basicKnowledge 由 `base/skill_index/executable/index.ts` 经 base 链解析（registry 不再持有），渲染/知识注入**逐字节不变**。

**关键 scope 决策（经双路 feasibility review 收敛）**：L4.1 **只接 renderXml + basicKnowledge**，**不碰 method 解析**（lookupMethodEntry/callMethod/renderMethodsNode）。理由：skill_index 无 method；method 沿链解析推迟到 **L4.2**（program 等 method 真搬进 base 时），那时再处理 `manager.refine`（同步方法）的 async 传播。这消掉「refine 同步无法 await」的 Critical，并大幅缩小 L4.1 的 async 面。

**Architecture:** 新增 `src/executable/windows/_shared/behavior.ts`（缓存 base registry + stat-before-import 加载 proto def + `resolveRenderXml`/`resolveBasicKnowledge`，async，链优先 registry 兜底）。热路径只改两处：render.ts 的 renderXml 取值、synthesizer 的 basicKnowledge 取值。`assertAllRenderHooksRegistered` 保持**同步**，用「chain-provided 声明集」让 skill_index（registry 移走 renderXml）通过。`onClose`/`compressView`/method 解析仍走 registry（L4 排除 / L4.2）。

**Tech Stack:** TypeScript / bun。复用 `src/executable/prototype/`（builtinProtoId/ObjectRegistry/ObjectRecord.dir）+ `src/extendable/base`（loadBuiltinRegistry）。

**权威**：L4 设计 spec §2/§3。基线（L4.0 后）：1055 pass / 0 fail / 3 skip，tsc 0 error。

## 设计决策

### D1 只接 renderXml + basicKnowledge（method 推迟 L4.2）
behavior.ts 提供 `resolveRenderXml(type)` + `resolveBasicKnowledge(type)`。L4.1 **不实现/不接** resolveMethod/resolveAllMethods——L4.2 转写 program 等 method 时再加（届时一并处理 `manager.refine` 同步→async）。`renderMethodsNode`（render.ts:78）/ `lookupMethodEntry`（manager.ts:82）/ `callMethod`（self.ts:31）**L4.1 不动**。

### D2 graceful dispatch（链优先、registry 兜底）
```
protoId = builtinProtoId(window.type)
chainHook = await resolveRenderXml(window.type)      // 链未提供该 aspect → undefined
renderXml = chainHook ?? getWindowTypeDefinition(window.type).renderXml
```
- skill_index：base/skill_index/有 executable 提供 renderXml → 走链。
- program/file/knowledge/command_exec/custom/search：base 骨架在 registry 但**无 executable** → 链 miss → 回退 registry（L4.2 加 executable 后自动切链）。
- do/todo/talk/plan/relation/feishu_*：不在 base registry → 回退 registry。

### D3 proto def 加载：stat-before-import（fail-loud，**不吞 broken import**）
动态 import 对「文件不存在」与「文件存在但内部 import 损坏」**同抛 `ERR_MODULE_NOT_FOUND`**（已实证）。故**先 stat `executable/index.ts`**：ENOENT → undefined（骨架，正常回退）；存在则 import，**任何 import 错误向上抛**（fail-loud，对齐 loader.ts:8-16 / object-record.ts 既有范式，遵守 silent-swallow ban）。

### D4 assertAllRenderHooksRegistered 保持同步 + chain-provided 声明集
skill_index registry 移走 renderXml 后需通过启动校验。**不 async 化**该函数（它是 windows/index.ts:94 顶层同步 boot 副作用，async 化 → fail-loud 退化为 unhandled rejection）。改为：windows/_shared/registry.ts 维护 `CHAIN_PROVIDED_RENDER: Set<WindowType>`，提供 `markRenderXmlViaPrototype(type)`；skill_index/index.ts 在注册时调它声明「renderXml 由原型链提供」。`assertAllRenderHooksRegistered` 判据改为 `def.renderXml || CHAIN_PROVIDED_RENDER.has(type)`——仍同步、仍 fail-loud（两者皆无的 type 仍 throw）。base proto 是否**真**提供 renderXml 由独立的「行为等价测试」+ 运行期 stat-before-import 兜底，不进 boot 同步路径。

### D5 onClose/compressView/method 不动（L4 排除 / L4.2）
close 钩子（manager.ts:361）、compressView（render.ts:155）、method 解析（manager/self/renderMethodsNode）**全保持 registry**。故 skill_index registry 入口**保留 onClose**，仅移走 renderXml + basicKnowledge。

### D6 缓存
base registry 由 `loadBuiltinRegistry()` 加载一次 module-level 缓存；proto def 按 dir 缓存。提供 `clearBehaviorCache()` 测试钩子（对齐 loader.ts:74 既有范式）。

```ts
// src/executable/windows/_shared/behavior.ts
import { stat } from "node:fs/promises";
import { join } from "node:path";
import { loadBuiltinRegistry } from "../../../extendable/base";
import { builtinProtoId, type ObjectRegistry } from "../../prototype";
import type { ObjectWindowDefinition } from "../../server/window-types";
import type { RenderHook } from "./registry";

let _registry: Promise<ObjectRegistry> | undefined;
function baseRegistry(): Promise<ObjectRegistry> {
  if (!_registry) _registry = loadBuiltinRegistry();
  return _registry;
}

const _defCache = new Map<string, Promise<ObjectWindowDefinition | undefined>>();
async function loadPrototypeDefinition(dir: string): Promise<ObjectWindowDefinition | undefined> {
  let p = _defCache.get(dir);
  if (!p) {
    p = (async () => {
      const file = join(dir, "executable", "index.ts");
      try {
        await stat(file);                       // 存在性探测：ENOENT→骨架
      } catch (e) {
        if ((e as NodeJS.ErrnoException).code === "ENOENT") return undefined;
        throw e;
      }
      const mod = await import(file);           // 存在则 import；import 错误向上抛（fail-loud）
      return (mod.window ?? undefined) as ObjectWindowDefinition | undefined;
    })();
    _defCache.set(dir, p);
  }
  return p;
}

/** 沿 extends 链 resolve 某 aspect；proto 不在 base registry 或链全 miss → undefined（回退 registry）。 */
async function resolveAspect<T>(
  type: string,
  pick: (def: ObjectWindowDefinition) => T | undefined,
): Promise<T | undefined> {
  const reg = await baseRegistry();
  const protoId = builtinProtoId(type);
  if (!reg.has(protoId)) return undefined;
  let curId: string | null = protoId;
  const seen = new Set<string>();
  while (curId !== null && !seen.has(curId)) {
    seen.add(curId);
    const rec = reg.get(curId);
    if (!rec) break;
    const def = await loadPrototypeDefinition(rec.dir);
    if (def) {
      const v = pick(def);
      if (v !== undefined) return v;
    }
    curId = rec.extends;
  }
  return undefined;
}

export async function resolveRenderXml(type: string): Promise<RenderHook | undefined> {
  return resolveAspect(type, (d) => d.renderXml);
}
/** basicKnowledge 是 string | ((ctx)=>string)；只取 string 形态（动态函数留 L4.2+）。 */
export async function resolveBasicKnowledge(type: string): Promise<string | undefined> {
  return resolveAspect(type, (d) => (typeof d.basicKnowledge === "string" ? d.basicKnowledge : undefined));
}

export function clearBehaviorCache(): void {
  _registry = undefined;
  _defCache.clear();
}
```
> 不复用 L2 同步 `resolveAlongChain`（probe 需 async load）——手动沿 `rec.extends` walk，同语义 + visited 防环（base registry build 已校验无环，这里 defense-in-depth）。
