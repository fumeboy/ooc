## 设计决策
### D1 resolveMethod / resolveAllMethods（扩 behavior.ts）
- `resolveMethod(type, name)`: resolveAspect(type, d => d.methods?.[name]) → 链命中 MethodEntry；miss→undefined。
- `resolveAllMethods(type)`: 沿链 root→own 合并（子覆盖父）；**链中无任何 proto 有 executable → 返回 undefined**（sawExecutable 标记），否则返回 merged（含 `{}`）。区分「已转写方法空」vs「骨架未转写→回退 registry 真方法」。

### D2 热路径 method dispatch（chain ?? registry）——**全部 method 消费点**
- `manager.lookupMethodEntry(parent, name)` → async：`(await resolveMethod(parent.type, name)) ?? getWindowTypeDefinition(parent.type).methods[name]`。
- `self.callMethod`：同（保留 custom 的 loadObjectWindow 分支——custom 链化是 L4.2 尾；resolveMethod("custom") 返 undefined（base/custom 无 executable）→ 落 custom 分支）。
- `render.renderMethodsNode` → async：`const m = (await resolveAllMethods(type)) ?? getWindowTypeDefinition(type).methods; Object.keys(m ?? {})`。
- **`permissions.ts`**（准入派生，读 `def.methods[command].permission`）→ chain-aware。**已核查链 async-ready**：`decidePermission`（:174 async）已被 `thinkloop.ts:371` await；`lookupDeclaredPermission`（:128 同步，:202 调）+ 其内 `tryWindow`（:157/:162）需改 async（唯一调用者 :202），传播终于 decidePermission（已 async）。改 `(await resolveMethod(type, command))?.permission ?? getWindowTypeDefinition(type).methods[command]?.permission`。
- **`synthesizer.ts:84` `lookupFormEntry`**（读 `def.methods[form.command]` 派生 form 的 method-knowledge：EXEC_KNOWLEDGE/OPEN_MATCH_KNOWLEDGE/EDIT_KNOWLEDGE 等）→ **必须 chain-aware**（C2）：trim 后否则 form 的 method 专属 knowledge 静默消失。改 `(await resolveMethod(parent.type, form.command)) ?? def.methods[form.command]`。caller `computeFormKnowledgeEntries`→`enrichFormCommandKnowledge`（:92）已 async-ready。
- **`api.list-window-types.ts:42-53`**（HTTP `/windows/_shared/types` catalog，读 def.methods + def.basicKnowledge）→ **chain-aware**（H2，避 UI catalog 丢 4 proto 方法/知识）：handler 改 async，用 resolveAllMethods + resolveBasicKnowledge ?? registry。

### D3 refine 同步→async 传播
`manager.refine(formId,args): boolean` → `async refine(...): Promise<boolean>`（内部 await lookupMethodEntry）。调用者：`command_exec/command.refine.ts`（exec 路径，已 async → 加 await）+ `__tests__/manager-refine-failed.test.ts`（`mgr.refine(...)` → `await mgr.refine(...)`，test 已 async）。grep 确认无其它同步调用者。

### D4 4 proto 转写（import-reuse，低漂移）
每 proto `base/<proto>/executable/index.ts`：`export const window: ObjectWindowDefinition = { methods: {<import 现有 method entries>}, renderXml: <import 现有 renderXml>, basicKnowledge: <import 现有 BASIC_KNOWLEDGE 若有> }`。method exec body 不 copy——从 `windows/<proto>/` import 现有 entry。exec ctx（MethodExecutionContext：thread/manager/args/self，字段皆 optional + body 防御性 null-check）不变，故 program.exec→runOneExec / search.open_match→manager.insertTypedWindow / file.edit 等依赖经同一 ctx 照常工作（exec body 无 module-level 可变状态依赖 import 位置——已核）。
- **H1：这些 const 当前多数未 export，base import 前须先加 `export`**（逐 proto）：
  - program：`execCommand`/`closeCommand`/`setHistoryWindowCommand` + `renderProgramWindow`（现 module-local，加 export）。无 basicKnowledge。
  - search：`closeCommand`/`openMatchCommand`（含 import 自 command.set-results-window 的 `setResultsWindowCommandForSearch`）+ `renderSearchWindow`（加 export）。`SEARCH_WINDOW_BASIC_KNOWLEDGE` 已 export。
  - file：5 entry（set_range/set_viewport/reload/close/edit）+ `renderFileWindow`（加 export）。无 basicKnowledge。
  - knowledge：3 entry（reload/close/set_viewport）+ `renderKnowledgeWindow`（加 export）。无 basicKnowledge。
- 各 `windows/<proto>/index.ts` registry 入口 **trim（移走 methods/renderXml/basicKnowledge），但按 C1 保留非-method/非-render 的 registry-served hook**：
  - **program → `{}`**（无 onClose/compressView）+ markRenderXmlViaPrototype。
  - **search → `{ compressView: compressSearchWindow }`**（search/index.ts:295）+ markRenderXmlViaPrototype。
  - **file → `{ compressView: compressFileWindow }`**（file/index.ts:473）+ markRenderXmlViaPrototype。
  - **knowledge → `{ onClose: onCloseKnowledgeWindow }`** + markRenderXmlViaPrototype。
  - （compressView/onClose 是 L4 排除项，仍 registry-served，render.ts:156 / manager.ts:361 不动——绝不能误 trim，否则 search/file 压缩态丢折叠渲染、knowledge close 拒绝失效。）
- self.md/readable.md 改 in-character（去 god-view；料源现有 BASIC_KNOWLEDGE/method knowledge）。
- **M2 注**：root 原型本轮**仍无 executable**——通用 close 暂各 proto 自带（不 hoist 到 root，保行为等价；root 收编通用方法是后续/L5-6）。

### D5 行为等价（每 proto 必证）
每 proto：method 调用结果 + renderXml 输出 + basicKnowledge 注入，转写前后**逐字节/等价**。靠 import-reuse（同函数引用）天然保证 + 等价测试守。
