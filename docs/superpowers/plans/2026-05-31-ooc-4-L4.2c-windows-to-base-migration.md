# OOC-4 L4.2c：windows→base 实现搬迁（skill_index/program/search/file/knowledge）

> 执行 sub-agent **不要自己 commit**。

**Goal:** 兑现用户方向「src/executable/windows 大部分程序搬迁到 src/extendable/base」——把 5 个已上链 entity 原型的 **proto 专属 behavior** 从 `windows/<proto>/` 物理 move 进 `base/<proto>/executable/`（反转 import：代码住 base）。windows/<proto>/ 退为**薄注册壳**（注册 L4 排除的 onClose/compressView + markRenderXmlViaPrototype）。**行为不变**（同代码搬家）。

**Architecture:** 反转 L4.1/L4.2 的 import-reuse。proto 专属代码（method entries / renderXml / basicKnowledge / onClose / compressView / proto-only helper）move 进 base；**跨 proto 共享 infra 留 executable**（base import 之）。ContextWindow 实例 **types.ts 留原位**（runtime 模型，被 manager/render/persistence 用，不入 extendable）。

**关键规则（shared-vs-specific 切分；经 feasibility review C1 修正）**：
- **搬进 base/<proto>/executable/**：method entries、renderXml、basicKnowledge、onClose、compressView 的**函数体**；+ `search/command.set-results-window.ts`（唯一真 proto-only helper，外部 importer 仅 behavior.test）。
- **留 executable（base import 之，因被 root 创建器命令用 = 跨域共享）**：
  - `windows/program/runtime.ts`（runOneExec：tools/wait.ts + root/command.program.ts 用）
  - `windows/program/history-viewport.ts`（DEFAULT_HISTORY_VIEWPORT：root/command.program.ts:26 用）
  - `windows/search/results-viewport.ts`（DEFAULT_RESULTS_VIEWPORT：root/command.glob.ts:26 + command.grep.ts:22 用）
  - base/<proto>/executable import 这些（extendable→executable，与 runtime 同向）。
- **留原位**：`windows/<proto>/types.ts`（ContextWindow 实例 type，被 _shared union/manager/render 经各自路径引用）。
- **薄壳** `windows/<proto>/index.ts`：import onClose/compressView from `base/<proto>/executable` + register + markRenderXmlViaPrototype。program 无 onClose/compressView→壳仅 register `{}` + mark。
- **C2 相对 import 改写**：搬进 base 的文件里凡 `./types.js`/`./results-viewport.js`/`./history-viewport.js`/`./runtime.js` 等指向「留在 windows」的相对 sibling import，须改成 `../../../../executable/windows/<proto>/<file>.js`（types/viewport/runtime 不随之搬）。

**循环：已 feasibility review 验证无环（H1）**——`behavior.ts → extendable/base/index → executable/prototype` 是 DAG sink（base/index 只 import prototype；prototype 不 import windows/_shared）；proto executable 由 behavior **动态 import**；`server/self.ts` 今天已静态 import behavior，故搬迁只加边不加新模块（bun build 确认无 cycle warning）。执行注意：base/<proto>/executable **不得**新增静态 import behavior.ts（保持 DAG sink）。
