## 本次迭代记录：session worktree 迁 flows/<sid> + HTTP 直 commit main + GC 保留运行时

### 动机

迭代 01 的 session worktree 物理落 `stones/session-<sid>/`（独立目录）；运行时数据（thread.json / .session.json / context.json 等）落 `flows/<sid>/`。两目录分离导致：① stone 读写与运行时隔两处，调试跟踪需关注两个目录；② `flows/<sid>` 与 worktree 生命周期不同步；③ GC 用 `gitWorktreeRemove --force` 会连 `flows/<sid>` 运行时数据一并清理，session 从前端凭空消失。

同期发现 thread.json 与 thread-context.json 双写漂移——self 门面窗落进 thread.json（6 window）但 thread-context.json 不含（5 window），legacy fallback 靠补漏维持，不是根本收口。

### 设计（方案 A，2026-06-09）

- **`flows/<sid>` 目录本身即 git worktree**：物理路径 = `flows/<sessionId>`，branch 名仍 `session-<sid>`（名路径解耦）。session 创建即 eager `git worktree add flows/<sid>` checkout main 全量文件，运行时产物靠 main `.gitignore` 白名单排出 git。
- **两类写两个落点**：LLM session 内任何 stone 写 → plain write 落 session worktree（不 commit main）；HTTP 控制面写 → `httpDirectMainWrite` 直 commit main，立即生效，不开 worktree。
- **GC 保留运行时**：`gitWorktreeUnregister` 删 `.git` link + prune 解除 worktree 身份，保留 `flows/<sid>` 目录中的运行时数据（thread 历史可回看）。
- **thread.json 不再含 self 门面窗**：`isNonPersistedWindow` = volatile derived 或 `isSelfWindow`，写盘时统一剔除，与 thread-context.json 端对齐。

### 实现（commits 0ab98a93 / c1085746 / 0610b774）

- `stone-worktree.ts:47` `sessionWorktreePath` 改返 `<baseDir>/flows/<sid>`；`ensureSessionWorktree`（L89）判 `.git` 判 worktree 已建（L75，非仅判目录存在），空目录 rmdir 后再 `gitWorktreeAdd`（L94-102）。
- `common.ts:79` `stoneDir` 新增 `session-` 分支路由：`flows/<sid>/objects/<id>/`（L81-93）。
- `versioning.ts:811` 新增 `httpDirectMainWrite`：直 commit main，无 worktree；`versioning.ts:405` `tryMergeSelf` ff-merge 后改调 `gitWorktreeUnregister`（L406）取代旧 `gitWorktreeRemove --force`。
- `context-window.ts:250` `isNonPersistedWindow` 加入 `isSelfWindow` 分支；`thread-json.ts:55` 引用此谓词统一过滤，self 门面窗不落 thread.json。

## 边界与未决

边界（不做什么）：

- `flows/<sid>/objects/` 与 `flows/<sid>/` 顶层运行时数据（threads / .session.json / context.json）共存同目录，物理上不分离——靠 `.gitignore` 白名单分离语义，不用目录结构区分。
- `gitWorktreeUnregister` 不删运行时数据，也不删 session git 分支（ff 后分支与 main 同 sha，`evolveSelfMerge` 负责删分支，`versioning.ts`：tryMergeSelf 只负责 unregister worktree）。
- HTTP 直 commit main 不触发 evolve_self 流程——人类已决策的编辑不走 session 隔离。

未决 / 跟进项：

- **abandoned worktree GC**（继承自迭代 01）：未 evolve 的 session worktree（`.git` link 还在）仅 evolve 路径回收；`pruneStaleWorktrees` 存在但默认仅启动 hygiene，长跑可能堆积 `.git` link。
- **thread-context 三文件收敛**（独立技术债，见设计文档 §1ter）：`thread.json.contextWindows[]` / `context.json` / `thread-context.json` 三处存储同一份 context；本次解决 self 门面漂移，完整退役 `thread.json.contextWindows[]` 仍是后续项（P5'.4 + §10）。
- **`flows/<sid>` `.gitignore` 维护**：main 分支根的 `.gitignore` 白名单新增 tracked 子目录时须同步更新，否则运行时产物污染 git status / evolve diff。
