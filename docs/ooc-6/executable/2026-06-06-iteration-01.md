# 执行（executable）能力维度 · 迭代 01（2026-06-06）

## 维度定位

Executable 是 Object 的行动能力维度：LLM 不直接调任意函数，只能通过稳定的 tool 原语在 ContextObject 上调 Method 来改变世界。锚 `packages/@ooc/meta/object.doc.ts:743`（节点 title「OOC Agent executable 概念」）。

核心分层（同 doc 757-762）：
1. Tool 原语层 — exec / close / wait / compress（4 个，LLM 直接可见）。
2. Method 层 — do / talk / program / plan / todo / end / open_file / write_file / glob / grep 等具体行动（2026-05-28 起 Command 与 Object Method 归一为 Method）。
3. ContextObject 层 — file/talk/program/do/plan 等上下文对象，每个背后是一个 OOC Object（builtin 或 user-defined）。
4. Registry / Manager 层 — 注册各 object type 的 method/readable/onClose/basicKnowledge。
5. Knowledge Activation 层 — 按 method path 自动激活执行所需知识。

## 本次迭代记录：write/edit/program-shell 收敛到 session worktree

**动机** — 旧 plain overlay 模型下，business session 改自己 identity（self.md / executable / 文件）写到 shadow overlay，存在「裸读看不到 overlay 改动」「program shell `OOC_SELF_DIR` 恒指 main 与 write_file 落点不一致」的死穴。需要让 Object 在一个 session 内对自身的所有读写收敛到单一、可裸读的目录。

**设计** — session identity = 一个 lazy 创建的 git worktree（`stones/session-<sid>/objects/<id>/`，main HEAD 的完整副本）。business session 对自己 identity 的读写都重定向到该 worktree（裸读裸写可见全量），改动落 worktree 不污染 main；只有 super flow 的 evolve_self 把整个 session 分支 commit→rebase→ff-merge 进 main 才永久。worktree 建失败一律 fall through 回 `versionedStoneWrite`，绝不裸写 main 绕过版本化。

**实现**
- program shell 通道（commit `726ab0e1`）：`buildProgramShellEnv` 改 async，`OOC_SELF_DIR` 经 `resolveStoneIdentityDir(ref, "write")` 解析 —— business session 指向 session worktree，super/控制面指向 main，取代旧 `stoneDir(deriveStoneFromThread())` 恒 main 写法（`packages/@ooc/core/executable/program/self-env.ts:16-28`）。
- write/edit/open 通道（commit `db9e54ea`）：file builtin write_file 自治区写重定向到 worktree（`packages/@ooc/builtins/file/executable/index.ts:596-638`，plain write 不走 versioning），失败 fall through `versionedStoneWrite`（:640）；`file_window.edit` / `open_file` 经 `resolveStoneWorktreeTarget`（同文件 :393）按 write/read 模式重定向（删除旧 `resolveStoneOverlayTarget`）。
- evolve_self 重做（同 commit）：不再逐文件应用 overlay，改为 commit creator session worktree → rebase → ff-merge main → GC（`packages/@ooc/core/programmable/evolve-self.ts`）。

**验证** — commit `db9e54ea` 改写全套测试为 worktree 模型并断言 business session 落点为 session worktree：`packages/@ooc/core/executable/__tests__/program.test.ts`（OOC_SELF_DIR 用例：ensureStoneRepo+commit main 后断言指向 worktree 目录）、`file-overlay-redirect.test.ts`、`root.command.write-file.versioning.test.ts`、`evolve-self.test.ts`。core + builtins 全绿，tsc / silent-swallow / deprecated gate 全绿。

## 边界与未决

- 边界：executable 不为每个能力新增 tool（tool surface 固定 4 个，新能力表现为 method 或 object type）；它只定义「如何行动」，不负责跨 Object 协作语义（归 collaborable）、不负责方法库形状与演化路径（归 programmable）、不负责前端渲染（归 visible）。
- compress tool 当前仅 `scope=windows` 落地，`scope=events / auto` 抛 not-implemented（`packages/@ooc/core/executable/tools/compress.ts`）。
- 自改 `stones/<self>/executable/index.ts` 当前没有硬 deny，仅靠 metaprog ask + write_file ask 的弱约束（doc Q0e 待补硬拦，`object.doc.ts:1080`）。
- executable 命令集 / 注册的 readable 是全局 main-canonical：per-session 改命令集本身不走 worktree 重定向，须经 evolve_self 合入 main 后重注册才生效。
