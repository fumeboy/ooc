## 本次迭代记录：建对象/改命令集统一经 session worktree + evolve_self + thread-context 双写漂移根治

### 动机

迭代 01 留有两处未统一：
1. **`create_object` 快捷命令**：建新对象走 `supervisorCreateObject`（`versioning.ts` 内）→ metaprog worktree → 独立 commit + merge，绕过 evolve_self 闸门，行为与改自己不对称，LLM 需区分两条路。
2. **thread.json 含 self 门面窗**：`isVolatileDerivedWindow` 过滤不含 self window，导致 thread.json 落盘含 self、thread-context.json 不含 self；legacy fallback 靠补漏维持，不够稳健，且 `readThread` 报 `references missing object`（self 不是 `_ref` 可解析的类型）。

### 设计

**建对象与改对象完全对称**：supervisor（或任何 Object）建新对象时，直接在业务 session 里 `write_file` 写 `stones/<newId>/{self.md,readable.md,...}` 到 session worktree，然后 super flow `evolve_self` 合入——cross-scope → PR-Issue → supervisor resolve，与改别人相同路径。不需要也不存在专用建对象命令。

**`versionedStoneWrite` 路径删除**：该函数封装了「open metaprog worktree → write → commit → merge」，是 HTTP 写和 `create_object` 的共同底层；两者分别改为 `httpDirectMainWrite`（HTTP 直 commit main）和「session worktree + evolve_self」后，`versionedStoneWrite` 无 caller，删除。

**thread.json 不再持久化 self window**：`isNonPersistedWindow` 加入 `isSelfWindow` 分支；self window 由 `initContextWindows` → `injectSelfWindowIfObjectThread` 每轮确定性重注入，无需持久化。

### 实现（commits 0ab98a93 / c1085746 / 0610b774）

- `versioning.ts` 删 `openMetaprogWorktree`（旧 L268）/ `supervisorCreateObject`（旧 L829）；删 `versioned-write.ts`（`versionedStoneWrite` 唯一 caller 已移除）。
- `index.ts:625` write_file 写重定向：去掉旧 `isOwnStone` 门槛，改为**任何** stone 自治区写（own + cross）都落 session worktree；以 `stoneClass.ownerObjectId`（目标对象）而非 authorObjectId 计算 worktree 路径（L633）；调 `resolveStoneIdentityRef({...targetObjectId}, "write")`（L653）。非业务 session 时 fail-loud（`sessionUsesWorktree` 判 L638）。
- `context-window.ts:250` `isNonPersistedWindow`：`return isVolatileDerivedWindow(window) || window.isSelfWindow === true`。
- `thread-json.ts:55` filter 改用 `isNonPersistedWindow`（取代旧 `isVolatileDerivedWindow`）。
- `evolve-self.ts:194` must-pr-issue 一等化（见 reflectable 迭代 02）。

## 边界与未决

边界（不做什么）：

- programmable 只描述*如何写才能生效*；路径权限 / 何时允许 cross-scope 合入由 reflectable.business_task_isolation + supervisor 治理决定。
- executable 命令集（`stones/<self>/executable/index.ts`）为全局 main-canonical：per-session 改命令集须经 evolve_self 合入 main 后重注册才生效；session worktree 里未 evolve 的命令不热更到 runtime。
- `versionedStoneWrite` 已删，HTTP 写由 `httpDirectMainWrite` 承接；write_file builtin（LLM 路径）不再有 versioned-write 语义包装，plain write 进 session worktree 即 stone 写的完整路径。

未决 / 跟进项：

- **命令集 session-aware 热更**：per-session 改 `executable/index.ts` 落 worktree 但未合入 main，该 session 内 `callMethod` 仍用 main canonical 版本；需 ServerLoader 感知 session worktree 才能 per-session 热更（loader.patches.watcher_driven_invalidation 升级项）。
- **thread-context 三文件收敛**（独立技术债）：self 窗漂移根治完成（本次），完整退役 `thread.json.contextWindows[]` 仍是后续项。
- **params schema 校验**（继承自迭代 01）：callCommand 路径 + ui callMethod 路径均缺自动参数检查/转换。
- **cross-scope 合入粒度**（继承自 reflectable 迭代 02）：整体分类不做文件级拆分（YAGNI）。
