# 元编程（programmable）能力维度 · 迭代 01（2026-06-06）

## 维度定位

programmable 描述 Object 持有并演化自身**自定义 ContextWindow + 命令表**的能力——在自己的 `stones/<self>/executable/index.ts` 里 `export const window`（type=custom 的 self window）+ 可选 `ui_methods`，LLM 通过统一 `exec(window_id="custom:<self>", command=...)` 调用，写文件即热更，是 OOC "自我塑造四件套"（reflectable/programmable/visible/readable）改方法的那一件。

子能力：自定义 self window（object_window_definition）/ 动态加载与热更（loader）/ ProgramSelf 注入（program_self_injection）/ custom 命令两条调用路径（custom_window_invocation）/ window 演化（window_evolution）/ metaprog·versioning 编排面。

锚：`packages/@ooc/meta/object.doc.ts:3686` 节点 `programmable`（title "OOC Agent programmable 概念"，含 5 个 children + 4 个 patches + todo）。

## 本次迭代记录

**主题：program shell `$OOC_SELF_DIR` 接入 worktree 统一模型（接续指南 §2 通道一）**

- **动机**：上一轮（commit 727a33a7，P1 路径收口）把 canonical stone 收到 `stones/main/objects/<id>/`，但 `OOC_SELF_DIR` 仍硬拼旧扁平路径 `stones/<id>/`，agent 用 program shell 写 `server/index.ts` 落到孤儿路径 → runtime 不加载 → call_method 恒 METHOD_NOT_FOUND（programmable harness S1/S2 均 Bad）。727a33a7 临时改用 `stoneDir(stoneRef)` 指向 main，但 worktree 模型落地后，business session 写 main 会污染所有 session；需让 program shell 与 write_file/edit 收敛到同一 session worktree 目录。

- **设计**：`OOC_SELF_DIR` 不再恒指 main，改由 `resolveStoneIdentityDir(ref, "write")` 解析——business session → 该 session worktree object 目录（`stones/session-<sid>/objects/<id>/`，main HEAD 完整副本，裸读裸写都看得到完整 identity，改动落 worktree 不污染 main，经 super flow evolve_self 合入才永久）；super / 控制面 → main canonical。program shell 由此与 write_file / edit 收敛到同一目录（统一模型五通道之通道一）。

- **实现**（commit `726ab0e1`，2026-06-06）：`packages/@ooc/core/executable/program/self-env.ts:22` 改 `buildProgramShellEnv` 为 async，调 `resolveStoneIdentityDir({baseDir,sessionId,objectId}, "write")`（`packages/@ooc/core/persistable/stone-worktree.ts:89`）取代旧 `stoneDir(deriveStoneFromThread())`；调用方 `packages/@ooc/builtins/program/executable/runtime.ts:65` 加 `await`。

- **验证**：`packages/@ooc/core/executable/__tests__/program.test.ts:67` 升级用例——先 `ensureStoneRepo` + commit identity 到 main，再断言 business session 的 `OOC_SELF_DIR` 指向 `${tempRoot}/stones/session-s1/objects/agent`（:88，真正覆盖新通道），并保留无 persistence 时 env UNSET 用例（:95）。tsc 干净 + executable 套件全绿。

## 边界与未决

边界（不做什么）：
- programmable 只描述 *如何写* 才能生效（mtime 热更条件），不规定 *谁可以写*——路径权限 / 是否允许 super flow 自动写由 reflectable.business_task_isolation + caller 显式请求决定。
- 不做新增 LLM tool 面；只给 Object "自我门面 window + 命令表"。
- params schema 校验当前不强制；custom window commands（LLM 路径）与 ui_methods（HTTP/agent-native 路径）形状不同、各写各的，不互相代替。
- knowledge / pool 数据文件操作不属本维度（persistable）；HTTP 写 stone 入口的 versioning 收口见 patches.http_writes_go_through_versioning。

未决 / 跟进项：
- params schema 校验未实现：若要自动参数检查/转换，需在 callCommand 路径 + ui callMethod 路径同时加（object.doc programmable.todo）。
- `export const window` 当前是单数；后续可演化为复数 windows 字典（注册多个自定义 window 类型）。
- hot-reload 仅 tier 1（fs watch → cache 失效）：tier 2 knowledge 增量 re-synthesis、tier 3 visible 浏览器端 HMR 仍 TODO（loader.patches.watcher_driven_invalidation）。
- 本次迭代修的是路径层根因；programmable tier=Bad 的完整闭环（引导 agent 用 write_file 而非 program shell 写 method、overlay × 自我编程交互）属 affordance/design，另议。
- mtime 失效假设 FS 毫秒精度；秒级精度 FS 有"写完立刻读旧版"极短窗口，无 etag/hash 兜底。
