## 一、前因后果（为什么做、依据什么）

- **设计权威**：`.ooc-world-meta/.../objects/supervisor/children/class/knowledge/object-model.md`（对象模型单一权威，核心 9 条）
  + 同目录 `example.md`（逐文件接口模板）。本轮任务＝把 `packages/@ooc/builtins/` 存量 class/object 改成与 example.md
  **完全一致**的接口模式，再反推 core。
- **核心转变**（旧 → 新）：
  - `self`：从「ContextWindow（id/class/title + 业务数据全平铺一坨）」→ 拆成纯 `Data`（types.ts）+ runtime 信封 + 独立投影态 `win`。
  - 方法：`exec(ctx)` 捆绑 `ctx.self/ctx.args` → `exec(ctx, self, args)` 三参分离；`self` 收窄为纯 Data。
  - 注册：副作用 `builtinRegistry.registerWindowClass(...)` barrel → 声明式 `index.ts` 的 `export const Class`。
  - readable：`(ctx)=>XmlNode[]` → `{ readable:(ctx,self,win)=>({class,content}), window:[{class,object_methods,window_methods}] }`。
  - constructor：返回 `{ok,window}` 的特殊 method → `Class.construct.exec(ctx,args)=>Data`（runtime 据 Data 包信封）。
  - package.json：去 `type`/`instantiate_with_new_world`/`members`，只留 `kind`(class/object)+`class`(继承)。
- **承重决策**（example.md 自陈「示意骨架非逐字」，矛盾/留白处由 supervisor 裁决，详见
  `docs/2026-06-15-object-model-migration.md` 第一节）：
  1. `self` = Data 本体（非 `{data}` 包装）；信封字段经 `ctx.object` 取。
  2. `types.ts` = 纯 Data；投影态 `win` 每 class 在 readable 自定义、与 Data 分离。
  3. constructor 取 `ConstructorContext`（无 `object`）、`exec(ctx,args)=>Data`、失败 throw；委托类 tool-object
     经 `ctx.runtime.instantiate('_builtin/<child>', args)`（`RuntimeHandle`）造子对象。
  4. **构造槽位键名是 `construct` 不是 `constructor`** —— JS `Object.prototype.constructor` 会遮蔽该键
     （`({}).constructor === Object` 恒真 → 单例无法识别；TS 也拿 `Function` 比对报错）。example.md 原写
     `constructor` 是该陷阱下的笔误，**已同步修正权威 example.md** + 全部落地代码。
  5. supervisor/user 按 object-model 迁移映射改为 **`kind:"object"`**（实例，非 class；`instantiate_with_new_world` 废弃）。

## 三、后续工作

### 3.1【主线】反推 core runtime 适配新契约（Wave 4，最大块）
1. **registry**（`runtime/object-registry.ts`）：`ObjectDefinition`(Record methods) → 存 `OocClass`，按 class id + 单链继承解析
   construct/executable/readable/persistable。**统一 class id 键名**（现 short「thread/plan」vs `_builtin/<id>` vs `_builtin/agent` 不一致）。
2. **loader**（`runtime/server-loader.ts`）：从 stone `index.ts` 的 `export const Class` 加载（替副作用 `export const window` barrel）。
3. **builtin 装载**：显式 import 各 `@ooc/builtins/<id>` 的 `Class` 注册进 registry（替 side-effect `registerWindowClass` on-import；
   root 旧 `import "@ooc/builtins/file"` 等已删，需新装载入口）。
4. **dispatch**：object method `exec(ctx,self,args)`（组装 `ExecutableContext` + 从 `OocObjectInstance` 取 `self=data`）；
   window method `exec(ctx,self,before_win,args)` → 写回新 `win`。
5. **投影**（`thinkable/context/renderers/xml.ts`）：调 `readable(ctx,self,win)` 渲染 `ReadableProjection {class,content}`。
6. **RuntimeHandle 实现**：`instantiate(classId,args)`（调 construct → Data 包成 `OocObjectInstance` → 挂进 thread）+ `close`。
   —— 委托 tool-object、agency plan/todo、pr 投递、plan.expand_step 全依赖它。
7. **constructor 调用**：`construct.exec(ctx,args)=>Data` → runtime 包信封（分配 id/class/title/status/createdAt）。
8. **data/win 持久化**：`OocObjectInstance {data, win}` 序列化（data 与 win 分离）；persistable 默认 save/load + per-class override。
9. **core 自有窗迁移**：`executable/windows/{talk, method_exec}` 迁新契约。**talk 是 thread/reflect_request 的父类**——
   它们的 readable（现为占位）+ say 待 talk 迁移后接回。
10. **bootstrap**：`app/server/bootstrap/instantiate-classes.ts` 移除 `instantiate_with_new_world`（supervisor 已 kind=object）。
11. **package.json 字段读写点**（`persistable/stone-object.ts` 等）：写 `kind`/`class`，去 `type`/`instantiate_with_new_world`。
12. **root 符号回流**：`core/executable/windows/index.ts` 停止 re-export 已删的 `ROOT_METHODS`(现为 array)/`getOpenableMethods`/
    `deriveRootIntentPaths`/`execRootMethod`；`core/executable/tools/exec.ts` 的 `getOpenableMethods()` 替代。
13. **feishu/lark executors**：`core/extendable/lark` 的 `executeOpenFeishuChat/Doc` 迁新契约（root 现用 bridge 适配，迁后删 bridge）。
