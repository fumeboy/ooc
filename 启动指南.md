# 启动指南

## 前置要求

1. **Go 1.24+**：确保已安装 Go 1.24 或更高版本
2. **Node.js 18+**：用于运行前端（如果使用前端）
3. **配置文件**：确保项目根目录下有 `.conf.xml` 配置文件

## 安装依赖

### 后端依赖

```bash
# 在项目根目录下运行
go mod download
go mod tidy
```

如果遇到 `golang.org/x/time/rate` 缺失的问题，运行：

```bash
go get github.com/labstack/echo/v4/middleware@v4.13.4
```

### 前端依赖

```bash
# 进入 web 目录
cd web

# 安装依赖
npm install
```

## 启动后端服务器

### 方式一：直接运行（推荐开发使用）

```bash
# 在项目根目录下
go run cmd/server/main.go
```

### 方式二：指定配置文件和端口

```bash
go run cmd/server/main.go -config .conf.xml -port 8080
```

### 方式三：编译后运行（推荐生产使用）

```bash
# 创建 bin 目录（如果不存在）
mkdir -p bin

# 编译
go build -o bin/server cmd/server/main.go

# 运行
./bin/server -config .conf.xml -port 8080
```

### 方式四：使用 Makefile（如果存在）

```bash
make run-server
```

### 验证后端启动

启动成功后，你会看到：

```
服务器启动在端口 8080
健康检查: http://localhost:8080/health
API 文档: http://localhost:8080/sessions
```

访问健康检查端点验证：

```bash
curl http://localhost:8080/health
```

应该返回：

```json
{
  "status": "ok",
  "time": "2024-01-01T12:00:00Z"
}
```

## 启动前端

### 开发模式

```bash
# 在 web 目录下
cd web
npm run dev
```

前端将在 `http://localhost:3000` 启动，并自动代理 API 请求到 `http://localhost:8080`。

### 生产模式

```bash
# 构建
cd web
npm run build

# 预览（可选）
npm run preview
```

## API 端点

### 基础端点

- `GET /health` - 健康检查
- `POST /api/sessions` - 创建会话
- `GET /api/sessions/:id` - 获取会话信息
- `POST /api/sessions/:id/ask` - 回答 Ask 问题
- `GET /api/sessions/:id/info/:info_id` - 获取信息对象详情

### 附身功能端点

- `POST /api/sessions/:id/possess` - 开始/停止附身
- `GET /api/sessions/:id/possess/request` - 获取附身请求
- `POST /api/sessions/:id/possess/respond` - 回复附身请求

## 测试 API

### 创建会话

```bash
curl -X POST http://localhost:8080/api/sessions \
  -H "Content-Type: application/json" \
  -d '{
    "user_request": "你好，请介绍一下你自己"
  }'
```

### 获取会话信息

```bash
curl http://localhost:8080/api/sessions/{session_id}
```

### 开始附身

```bash
curl -X POST http://localhost:8080/api/sessions/{session_id}/possess \
  -H "Content-Type: application/json" \
  -d '{
    "possess": true
  }'
```

## 环境变量

可以通过环境变量覆盖配置文件中的设置：

- `OOC_AI_PROVIDER` - AI 提供商
- `OOC_AI_API_KEY` - API 密钥
- `OOC_AI_BASE_URL` - API 基础 URL
- `OOC_AI_MODEL` - 模型名称
- `OOC_AI_MAX_TOKENS` - 最大 token 数
- `OOC_AI_TIMEOUT` - 超时时间（秒）

示例：

```bash
export OOC_AI_API_KEY="your-api-key"
go run cmd/server/main.go
```

## 常见问题

### 1. 端口被占用

如果 8080 端口被占用，可以指定其他端口：

```bash
go run cmd/server/main.go -port 8081
```

同时需要修改前端的代理配置（`web/vite.config.js`）中的 `target`。

### 2. 配置文件不存在

确保项目根目录下有 `.conf.xml` 文件，或使用 `-config` 参数指定配置文件路径。

### 3. 前端无法连接后端

检查：
- 后端是否正常启动
- 前端代理配置是否正确（`web/vite.config.js`）
- 浏览器控制台是否有 CORS 错误（后端已配置 CORS 中间件）

### 4. 优雅退出

按 `Ctrl+C` 可以优雅关闭服务器，服务器会等待最多 10 秒来完成正在处理的请求。

## 开发建议

1. **后端开发**：使用 `go run` 进行快速迭代
2. **前端开发**：使用 `npm run dev` 启用热重载
3. **调试**：查看服务器日志了解请求处理情况
4. **测试**：使用 `curl` 或 Postman 测试 API
